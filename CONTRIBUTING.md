# Contributing to OpenImageLab

Thank you for your interest in contributing to **OpenImageLab**!  
Our mission is to help everyone repair, enhance, and understand digital images using open-source tools.  
Contributions of all kinds are welcome: tutorials, documentation, code improvements, bug reports, and community support.

---

## How to Contribute

### 1. Reporting Issues
If you encounter a problem or spot an error (in documentation, examples, or code):

- Go to **Issues** → **New Issue**.
- Provide a clear title and description.
- Include steps to reproduce the problem if applicable.
- Add screenshots or example images if relevant.

### 2. Creating or Improving Tutorials
If you want to add a tutorial:

- Create a new folder in `tutorials/`.
- Add a markdown file named:  
  `YYYY-MM-topic-title.md`  
  (example: `2025-01-image-brightness-correction.md`)
- Include:
  - The objective of the tutorial
  - Step-by-step instructions
  - Input and output image examples (if possible)
  - Links to referenced free/open-source software

### 3. Submitting Pull Requests
1. Fork the repository.  
2. Create a new branch:  
   `feature/tutorial-name` or `fix/issue-description`
3. Make your changes (tutorials, docs, examples, code).  
4. Submit a Pull Request with:
   - A clear description
   - A reference to related issues (if any)

### 4. Writing Style Guidelines
- Prefer clarity over complexity.
- Use simple, neutral English.
- Use Markdown headings to structure content.
- When explaining processing steps, include:
  - Before/After comparison
  - Software used
  - Settings or parameters if applicable

### 5. Code of Conduct
We maintain a respectful and inclusive environment.  
Harassment, discrimination, or hostile behavior of any kind will not be tolerated.

### 6. License
All contributions are published under the repository’s license unless explicitly stated otherwise.

Thank you for helping make OpenImageLab a valuable resource for everyone!
